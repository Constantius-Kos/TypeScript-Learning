## 📝 Ключові інсайти по урокам

### Урок 8: Type Aliases
- `type` гнучкіший за `interface` — примітиви, union, tuple — тільки через `type`
- **Єдина точка зміни:** змінив alias → оновилося у всіх місцях
- `type Status = string | number` — валідний union з різних типів
- Можна комбінувати: `readonly` + `?` + aliases в одному об'єкті

### Урок 7: Readonly властивості
- `readonly` це захист на етапі **розробки** від помилок самих розробників
- Класика: конфіги БД (host, port) задаються один раз — `readonly` страхує від опечаток
- `readonly string[]` — масив тільки для читання, не можна `push`/`pop`

### Урок 6: Optional властивості
- `property?: Type` — властивість може бути відсутньою
- `?.` — optional chaining для безпечного доступу
- `??` — nullish coalescing, альтернатива для `null`/`undefined`

### Урок 5: Union & Literal types
- `type A = "x" | "y"` — тільки конкретні значення
- Union дозволяє змінній бути одним з кількох типів
- Literal types роблять API самодокументованим

### Урок 4: Type inference
- TS сам визначає тип зі значення — не завжди потрібно писати анотацію
- Але явна анотація = краща документація та захист

### Урок 3: Interfaces
- `interface` описує структуру об'єкта
- Використовуй `interface` для об'єктів, `type` для всього іншого

### Уроки 1-2: Примітиви, масиви, tuple
- `string`, `number`, `boolean` — базові типи
- `number[]` або `Array<number>` — масив
- `[string, number]` — tuple (фіксована структура)
