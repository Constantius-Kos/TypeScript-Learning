## 📝 Ключевые инсайты по урокам

### Урок 8: Type Aliases
- `type` гибче `interface` — примитивы, union, tuple — только через `type`
- **Единая точка изменения:** поменял alias → обновилось во всех местах
- `type Status = string | number` — валидный union из разных типов
- Можно комбинировать: `readonly` + `?` + aliases в одном объекте

### Урок 7: Readonly свойства
- `readonly` это защита на этапе **разработки** от ошибок самих разработчиков
- Классика: конфиги БД (host, port) задаются один раз — `readonly` страхует опечатки
- `readonly string[]` — массив только для чтения, нельзя `push`/`pop`

### Урок 6: Optional свойства
- `property?: Type` — свойство может отсутствовать
- `?.` — optional chaining для безопасного доступа
- `??` — nullish coalescing, альтернатива для `null`/`undefined`

### Урок 5: Union & Literal types
- `type A = "x" | "y"` — только конкретные значения
- Union позволяет переменной быть одним из нескольких типов
- Literal types делают API самодокументируемым

### Урок 4: Type inference
- TS сам определяет тип из значения — не всегда нужно писать аннотацию
- Но явная аннотация = лучшая документация и защита

### Урок 3: Interfaces
- `interface` описывает структуру объекта
- Используй `interface` для объектов, `type` для всего остального

### Уроки 1-2: Примитивы, массивы, tuple
- `string`, `number`, `boolean` — базовые типы
- `number[]` или `Array<number>` — массив
- `[string, number]` — tuple (фиксированная структура)
