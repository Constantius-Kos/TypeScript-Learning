# 1. 📌 Тема уроку: Optional і default параметри функцій

## 2. 🎯 Ціль
Після цього уроку ти зможеш робити параметри функцій необов'язковими (`?`) і задавати їм значення за замовчуванням (`= значення`). Ти будеш розуміти, коли і навіщо застосовувати кожен із цих підходів.

## 3. 🤔 Проблема в JavaScript

В JS функцію можна викликати з меншою кількістю аргументів, ніж вона очікує — і нічого не відбудеться (поки не станеться баг):

```javascript
// JavaScript
function createUser(name, role, age) {
  console.log(`${name}, роль: ${role}, вік: ${age}`);
}

createUser("Алекс");
// Виведе: "Алекс, роль: undefined, вік: undefined"
// Жодних помилок! Просто тихий баг.
```

Розробники в JS зазвичай захищалися так:

```javascript
function createUser(name, role, age) {
  role = role || "user";       // якщо не передали — ставимо "user"
  age = age !== undefined ? age : 18; // захист через перевірку
}
```

Це працює, але виглядає як саморобний милиця. TypeScript дає нам вбудовані інструменти для цього.

---

## 4. 📖 Теорія

### Optional параметри (`?`)

Ставиш `?` після імені параметра — і він стає необов'язковим. TypeScript буде знати, що його можуть не передати.

```typescript
function greet(name: string, greeting?: string): void {
  //                                       ↑
  //               знак питання = "цей параметр необов'язковий"
}
```

**Важливо:** всередині функції тип такого параметра буде `string | undefined`. Це тому, що якщо його не передали — там `undefined`. TypeScript чесно говорить тобі про це.

**Правило:** обов'язкові параметри завжди йдуть ПЕРШИМИ, optional — в кінці. Інакше TS видасть помилку.

```typescript
// ✅ Правильно
function fn(required: string, optional?: number): void {}

// ❌ Неправильно — обов'язковий після необов'язкового
function fn(optional?: number, required: string): void {}
```

---

### Default параметри (`= значення`)

Інший спосіб — задати значення за замовчуванням прямо в сигнатурі:

```typescript
function greet(name: string, greeting: string = "Привіт"): void {
  //                                              ↑
  //                          якщо не передали — використовується "Привіт"
}
```

**Ключова відмінність від `?`:**
- `greeting?: string` → тип всередині функції: `string | undefined` (потрібно перевіряти!)
- `greeting: string = "Привіт"` → тип всередині функції: завжди `string` (вже безпечно!)

**Чому default краще в більшості випадків?** Тому що ти одразу отримуєш готове значення і не зобов'язаний писати `if (greeting !== undefined)` всередині тіла функції.

---

### Коли що використовувати?

| Ситуація | Що використовувати |
|---|---|
| Параметр може не передаватися, і це нормально — ти обробиш `undefined` | `?` (optional) |
| Параметр може не передаватися, але у нього є розумне значення за замовчуванням | `= value` (default) |
| Параметр завжди обов'язковий | нічого — звичайний параметр |

## 5. 💡 Приклади з детальними коментарями

### Приклад 1: Optional параметр

```typescript
// greeting? означає: можеш передати, можеш не передавати
function sayHello(name: string, greeting?: string): string {
  // Всередині greeting має тип: string | undefined
  // Тому нам потрібна перевірка перед використанням:
  if (greeting !== undefined) {
    return `${greeting}, ${name}!`;
  }
  return `Привіт, ${name}!`; // fallback якщо greeting не передали
}

sayHello("Алекс");            // "Привіт, Алекс!"
sayHello("Алекс", "Добрий день"); // "Добрий день, Алекс!"
```

### Приклад 2: Default параметр

```typescript
// Якщо currency не передали — буде "UAH"
function formatPrice(price: number, currency: string = "UAH"): string {
  // currency тут завжди string — жодного undefined!
  // Не потрібно перевіряти — TypeScript гарантує значення
  return `${price} ${currency}`;
}

formatPrice(1000);         // "1000 UAH"
formatPrice(50, "USD");    // "50 USD"
```

### Приклад 3: Реальний use case — створення користувача

```typescript
type UserRole = "admin" | "editor" | "viewer";

type User = {
  name: string;
  role: UserRole;
  age: number;
};

// role за замовчуванням "viewer", age — необов'язковий
function createUser(name: string, role: UserRole = "viewer", age?: number): User {
  return {
    name,
    role,
    age: age ?? 0, // якщо age не передали (undefined), використовуємо 0
    //      ↑↑
    //  nullish coalescing — пам'ятаєш з Уроку 6?
  };
}

const user1 = createUser("Алекс");              // role="viewer", age=0
const user2 = createUser("Марія", "admin");     // role="admin", age=0
const user3 = createUser("Влад", "editor", 25); // role="editor", age=25
```

## 6. 🔍 JS vs TS порівняння

**Було в JS (саморобний захист):**
```javascript
function sendEmail(to, subject, body, cc) {
  cc = cc || [];           // Милиця: якщо не передали — порожній масив
  subject = subject || "Без теми"; // ще одна милиця
  // ...
}
```

**Стало в TS (чисто і явно):**
```typescript
function sendEmail(
  to: string,
  subject: string = "Без теми",
  body: string,
  cc: string[] = []
): void {
  // Все чисто, жодних милиць. TypeScript сам стежить.
}
```

Переваги:
- Одразу видно які параметри необов'язкові прямо з сигнатури
- Немає болючого `|| []` який ламається на `0` або `false`
- IDE автодоповнює і підказує значення за замовчуванням

## 9. 🛠️ Практична порада

В реальних проектах **default параметри зустрічаються набагато частіше** ніж optional. Причина проста: якщо параметр необов'язковий — зазвичай вже є розумне значення за замовчуванням. Писати `= значення` чистіше і безпечніше, ніж потім перевіряти `if (param !== undefined)` всередині функції.

Optional `?` частіше використовують коли параметр справді факультативний і у нього немає жодного "розумного за замовчуванням" — наприклад, callback-функція або додаткові налаштування.
