# Нотатки — Урок 1: Класи в TypeScript

## Питання та відповіді

### Конструктор — що це?
Конструктор — спеціальний метод який автоматично викликається при `new ClassName()`.
Він отримує аргументи які передаються при створенні об'єкта і присвоює їх властивостям.

```ts
const user = new User("Алекс", 25);  // "Алекс" і 25 йдуть в конструктор
```

Якщо передати більше/менше аргументів — TS одразу дає помилку:
```
Expected 2 arguments, got 1
```
- **Expected** [іkспе́ктед] — очікувалось (від *expect* [іkспе́кт] — очікувати)
- **got** [ґот] — отримано (минулий час від *get* [ґет] — отримати)

---

### Навіщо `public` в конструкторі?
`public` в конструкторі — це скорочення (синтаксичний цукор).
TypeScript автоматично розгортає його у повний запис:

```ts
// Скорочено:
constructor(public name: string) {}

// TS розуміє як:
name: string;
constructor(name: string) {
  this.name = name;
}
```

Без `public` — параметр існує тільки всередині конструктора і не стає властивістю класу.

`public` — значення за замовчуванням. Якщо не писати нічого — властивість і так публічна.
Протилежності: `private`, `protected` — Урок 2.

---

### Навіщо `this` в методах?
`this` — посилання на поточний екземпляр об'єкта ("я сам").
Дає доступ до всіх властивостей та методів цього об'єкта:

```ts
class User {
  constructor(public name: string) {}

  greet(): string {
    return "Привіт, " + this.name;  // this = конкретний об'єкт
  }
}

const user1 = new User("Алекс");  // this.name = "Алекс"
const user2 = new User("Боб");    // this.name = "Боб"
```

Без `this` — JS шукає змінну `name` в локальній області і не знаходить її.

`this` дає доступ не тільки до властивостей, але й до інших методів класу:
```ts
this.deposit(amount);   // виклик іншого методу
this.balance += amount; // доступ до властивості
```

---

## Ключові інсайти

- Класи в TS = класи в JS + типізація властивостей та методів
- `public` в конструкторі — просто зручна домовленість розробників TS
- `this` = "цього об'єкта" — доступ до всього всередині об'єкта
- Вчиш TypeScript → автоматично повторюєш JS класи
